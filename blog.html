1. Discuss the scope of var, let, and const ?

Var:-

  • Function Scope: Variables declared with var are function-scoped. This means they are only accessible within the function in which they are declared or globally if declared outside of any function.
  • Hoisting:Variables declared with var are hoisted to the top of their containing function or global scope. This means you can access a var variable before it's declared in code, but it will be undefined.
  • No Block Scope: Unlike let and const, var does not have block-level scope. Variables declared with var inside a block (like a loop or an if statement) are visible throughout the entire function. 

                              function example() {
                                  if (true) {
                                      var x = 10;
                                  }
                                  console.log(x); // x is accessible here, and its value is 10
                              }

Const:-

  • Block Scope:Like let, const variables are block-scoped.
  • No Hoisting:const variables are not hoisted, and trying to access them before they're declared results in a ReferenceError.
  • Immutable: const variables are constant, meaning their value cannot be reassigned after they are initialized. However, if a const variable holds an object or an array, the properties or elements of the object or array can still be modified. 

                              if (true) {
                                  let y = 20;
                              }
                              console.log(y); // ReferenceError: y is not defined

Let:-

  • Block Scope: Variables declared with let are block-scoped. They are only accessible within the block (enclosed by curly braces) in which they are defined.
  • No Hoisting: Unlike var, let variables are not hoisted to the top of their containing block. Trying to access them before they're declared results in a ReferenceError. 

                              const z = 30;
                                  z = 40; // Error: Assignment to a constant variable
  
                              const myArray = [1, 2, 3];
                              myArray.push(4); // This is allowed
                              myArray = [4, 5, 6]; // Error: Assignment to a constant variable

Tell us the use cases of null and undefined ?

In JavaScript, undefined means a variable has been declared but has not yet been assigned a value. On the other hand, null is an assignment value that represents no value or no object. It is often used to represent an intentional absence of any object value, as when a function returns null but there is no object for it to return. some use cases of null and undefined:

  • Use undefined when you want to check if a variable has been initialized or not.
  • Use null when you want to indicate that a variable has no value.
  • Use undefined when you want to check if an object property exists.
  • Use null when you want to indicate that an object property has no value.

3.What do you mean by REST API?

A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style. It is an application programming interface (API) that allows for interaction with RESTful web services. REST stands for representational state transfer and was created by computer scientist Roy Fielding12. REST APIs provide a flexible, lightweight way to integrate applications and to connect components in microservices architectures1. They are used to access and manipulate web resources using a uniform and predefined set of stateless operations1.
